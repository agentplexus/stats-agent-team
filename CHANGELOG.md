## [Unreleased]

## [0.5.0] - 2026-01-04

### Added

- Phoenix observability provider support for LLM tracing
- `HTTP_TIMEOUT_SECONDS` environment variable for configurable HTTP client/server timeout
- `pkg/logging` package for centralized structured logging with slog
- Provider-specific observability configuration (PHOENIX_API_KEY, PHOENIX_SPACE_ID)

### Changed

- All agents refactored to use structured logging package
- HTTP servers now use configurable timeouts from config
- OmniLLM adapter updated to pass timeout configuration

### Dependencies

- `github.com/agentplexus/omnillm` v0.9.0 → v0.10.0
- `github.com/agentplexus/go-phoenix` v0.1.0 (new)
- `github.com/agentplexus/go-opik` v0.4.0 → v0.5.0
- `github.com/agentplexus/omniobserve` v0.4.0 → v0.5.0
- `github.com/cloudwego/eino` v0.7.14 → v0.7.17
- `github.com/agentplexus/omniserp` v0.6.0 → v0.7.0

## [0.4.0] - 2025-12-27

### Added

- Helm chart for Kubernetes deployment with Minikube and EKS support
- Parameterized Dockerfile (`Dockerfile.agent`) for building all agent images
- CI/CD pipeline with Gitleaks, Trivy, Kubeconform, and Polaris
- Go struct validation for Helm values (`pkg/helm/`)
- Helm chart unit tests with helm-unittest
- MkDocs documentation site with Material theme

### Changed

- **BREAKING:** Module moved from `github.com/grokify/stats-agent-team` to `github.com/agentplexus/stats-agent-team`
- **BREAKING:** Dependencies renamed: metallm → omnillm, metaobserve → omniobserve, metaserp → omniserp

### Fixed

- Unchecked error return value in Helm validation registration

## [0.3.0] - 2025-12-21

### Added

- A2A Inter-Agent Protocol support for all agents
- LLM Observability via MetaObserve (Opik, Langfuse, Phoenix backends)
- MetaLLM adapter replacing gollm adapter
- Security documentation (`README_SECURITY.md`)
- Project roadmap (`ROADMAP.md`)
- Presentation materials (`presentation.md`)

### Changed

- Enhanced team stats finding performance via `make run-all-eino`

### Dependencies

- `github.com/cloudwego/eino` 0.7.8 → 0.7.10
- `github.com/a2aproject/a2a-go` (new)
- `github.com/grokify/metallm` (new)
- `github.com/grokify/metaobserve` (new)

## [0.2.0] - 2025-12-14

### Added

- Synthesis Agent for LLM-powered statistic extraction from web content
- Complete 4-agent architecture (Research, Synthesis, Verification, Orchestration)
- Docker support with Dockerfile, docker-compose.yml, and entrypoint script
- Serper/SerpAPI integration for real web search
- Multi-LLM provider support (Gemini, Claude, OpenAI, xAI Grok, Ollama)
- Direct LLM search mode with `--direct` CLI flag
- Base agent package (`pkg/agent/base.go`)

### Changed

- **BREAKING:** Removed pre-built binaries from `bin/` directory
- **BREAKING:** Research agent API changes for 4-agent architecture compatibility

### Fixed

- Search functionality issues
- LLM support for synthesis and verification agents

### Dependencies

- `github.com/grokify/gollm` (new)
- `github.com/grokify/metasearch` (new)

## [0.1.0] - 2025-12-13

### Added

- Initial release with 3-agent architecture (Research, Verification, Orchestration)
- Research Agent with Google ADK and Gemini 2.0 Flash
- Verification Agent for source validation
- ADK Orchestration Agent with LLM-based coordination
- Eino Orchestration Agent with deterministic graph-based workflow
- MCP Server integration for Claude Code
- Multi-LLM provider support (Gemini, Claude, OpenAI, Ollama)
- Structured JSON output for verified statistics
- GitHub Actions CI/CD with golangci-lint
